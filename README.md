# QnA PDF Assistant

FastAPI backend + static web UI: upload PDFs, ask questions (Groq), multi-document search, JWT auth.

## Repository layout (this folder = Git root)

```
.
├── Dockerfile           # Production container (Render / Docker)
├── render.yaml          # Render Blueprint (optional one-click)
├── docker-compose.yml   # Local Docker + SQLite volume
├── requirements.txt
├── app.py               # Local dev: uvicorn with reload
├── backend/             # FastAPI app
│   ├── main.py
│   ├── auth.py
│   ├── config.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   ├── routers/
│   └── services/
└── static/              # Frontend (served by FastAPI)
    ├── index.html
    └── app.js
```

## Deploy on Render

1. **Recommended:** Initialize Git **inside this folder** and push so the repo root contains `Dockerfile`, `backend/`, and `static/` (no extra parent directory).
2. If the remote repo is a **monorepo**, set Render **Settings → Root Directory** to this folder (e.g. `Qna`) so `./Dockerfile` resolves correctly.
3. **Dashboard → New → Blueprint** → connect the repo → Render reads `render.yaml`.
4. Set **`GROQ_API_KEY`** in the dashboard (required). **`SECRET_KEY`** is auto-generated by the blueprint unless you override it.
5. Open the service URL; SQLite lives on the **disk** mounted at `/app/data`.

If the blueprint fails on **free** plan (e.g. disk not available), create a **Web Service** manually: **Environment = Docker**, root directory = repo root, add the same env vars and a **persistent disk** at `/app/data` with `DATABASE_URL=sqlite:////app/data/qna.db`.

Details: [DEPLOY.md](./DEPLOY.md).

## Run locally

```bash
cp .env.example .env   # edit SECRET_KEY and GROQ_API_KEY
pip install -r requirements.txt
python app.py
```

Or: `docker compose up --build`.
